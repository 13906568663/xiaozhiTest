# 闪购外卖派单模拟系统(独立演示项目)

模仿淘宝闪购风格的外卖派单管理后台 + MCP 工具,给「外卖骑手 AI 语音助手」演示用:
AI 通过 MCP 访问派单系统的订单,结合跑单业务做接单、取货顺序、路线规划决策。

**这是一个完全独立的项目**:自带依赖和虚拟环境,不 import 主项目任何代码、不碰主项目数据库。
不需要时直接删除本目录,并在 agent-flow 能力注册表里删掉对应 MCP 能力即可,主项目零残留。

## 模拟设定

- 固定种子数据:**5 家店铺、1 名骑手(小郑)、3 位买家**,分布在 5km × 5km 虚拟地图上;
- 订单 = N 个取货点 + 1 个送达点,三种典型单型:
  - **餐饮出餐单**:商家备货 8~15 分钟,骑手早到会"干等"(等待时长计入统计);
  - **即取单(充电宝柜机)**:备货 0 分钟,到店即取;
  - **跨店多取单**:同一订单要跑 A、B 两家店取货,全部取齐才能配送;
- 时间可加速:默认 5 倍速(现实 1 秒 = 模拟 5 秒),可冻结,演示不用干等出餐;
- 数据全内存,重启或点"重置数据"即恢复初始状态。

## 快速开始

```bash
cd shangou-simulator
uv sync
uv run python main.py
```

- 管理后台:http://127.0.0.1:18100/
- MCP 接入点(streamable-http):`http://127.0.0.1:18100/mcp`

环境变量:`SIM_HOST`(默认 127.0.0.1,对外部署改 0.0.0.0)、`SIM_PORT`(默认 18100)、
`SIM_TIME_SCALE`(默认 5)。

冒烟测试(先启动服务):`uv run python scripts/smoke.py`

## 接入 agent-flow(主项目)

1. 平台「能力注册 → MCP」新增:transport 选 `streamable_http`,URL 填 `http://<部署机>:18100/mcp`;
2. 把该能力绑定到"骑手语音助手"机器人;
3. 车机语音 → 小智链路 → 机器人 → 自动可用下面 6 个工具。

### MCP 工具清单

| 工具 | 说明 |
|---|---|
| `list_orders` | 在途订单列表(含各取货点备货倒计时、骑行分钟数) |
| `get_order_detail` | 单个订单详情 + 两两点位骑行时间矩阵(路线规划用) |
| `accept_order` / `reject_order` | 接单 / 拒单 |
| `update_order_status` | 到店 / 取货 / 送达(早到会返回需等待分钟数;跨店单需带 shop_id) |
| `get_rider_stats` | 今日单量、收入、超时单、累计干等分钟 |

### 机器人提示词建议(片段)

> 你是外卖骑手的语音助手。回答要短、口语化。涉及订单时先调工具查实时数据,
> 不要凭记忆。规划路线时综合考虑:各取货点备货倒计时(避免早到干等)、
> 点位间骑行时间、各订单送达时限;给出明确的先后顺序和理由,一次说清。

## 演示台本(客户验收场景)

1. 管理后台点 **「一键生成演示场景」**:同时出 3 张单——跨店单(鲜果店+便利店)、
   充电宝即取单、麻辣烫出餐单(12 分钟出餐);
2. 骑手对车机说:"有没有新单?" → AI 报三张单的概况和收益;
3. "都接了,先送哪个?" → AI 给顺序:先取充电宝(不用等)→ 顺路跨店取货 → 麻辣烫
   最后去正好出餐,不用干等;
4. 骑手按 AI 建议逐步上报"到店/取到了/送完了",后台大屏实时看到订单流转、
   骑手位置移动和 AI 的每一次工具调用(事件流紫色标签);
5. "今天跑了多少?" → AI 报单量、收入、超时和干等统计。

演示技巧:讲解时点「⏸ 冻结」暂停模拟时间;想跳过等待就切 10x。

## 目录结构

```text
shangou-simulator/
├── main.py               # 入口:FastAPI(管理后台+REST) + 挂载 MCP
├── scripts/smoke.py      # 端到端冒烟测试
└── shangou_sim/
    ├── clock.py          # 可加速/冻结的模拟时钟
    ├── store.py          # 内存数据层:种子数据、订单状态机、事件流
    ├── mcp_tools.py      # MCP 工具(streamable-http)
    ├── api.py            # 管理后台 REST API
    └── static/           # 管理后台页面(无依赖原生 JS)
```
