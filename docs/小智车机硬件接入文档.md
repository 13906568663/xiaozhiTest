# 小智车机接入说明

> 自建语音服务端，供硬件/固件对接。只讲两件事：**怎么接入** + **怎么测主动播报**。
> 协议与小智官方 ESP32 固件完全一致，不用改协议栈。

---

## 一、服务端地址

| 用途 | 地址 |
|---|---|
| OTA 配置接口 | `http://186.244.238.6:8989/xiaozhi/ota/` |
| WebSocket 接入 | `ws://186.244.238.6:8989/xiaozhi/v1/` |
| 主动推送接口 | `POST http://186.244.238.6:8989/admin/inject_msg` |

---

## 二、设备怎么接入

1. **改 OTA 地址**：把固件的 OTA 服务器地址改成
   `http://186.244.238.6:8989/xiaozhi/ota/`
   - 方式一：设备 AP 配网页（`192.168.4.1`）里填；
   - 方式二：编译时改配置项 `CONFIG_OTA_URL`。
   其余（WebSocket 地址等）由 OTA 接口自动下发，不用写死。

2. **设备标识**：用设备 MAC 作为 `Device-Id`（小写冒号分隔，如 `aa:bb:cc:dd:ee:01`），每台唯一——服务端靠它定点推送。

3. 连接流程与官方固件一致：OTA 拿配置 → WebSocket 握手（`hello`，opus / 16000Hz / 单声道 / 60ms 帧）→ 音频与信令收发。

### 实现主动播报的三个硬性要求（重点）

- **开机即连、长连接常驻**：连上后**不要因为一轮对话结束就断开**；
- **断线自动重连**：网络恢复后无限次 + 退避重连；
- **待机可收下发**：空闲时收到服务端下发的 `tts start → 音频帧 → tts stop`，要正常解码播放。

> 没有常驻长连接，主动播报就推不到——服务端找不到离线设备。

---

## 三、怎么测主动播报

设备接入并保持在线后，任意机器执行下面命令（把 `device_id` 换成目标设备的 MAC）：

```bash
curl -X POST "http://186.244.238.6:8989/admin/inject_msg" \
  -H "Content-Type: application/json" \
  -d '{"device_id":"aa:bb:cc:dd:ee:01","message":"您已超速，请减速慢行","skip_llm":true,"auto_listen":false}'
```

参数说明：

| 参数 | 说明 |
|---|---|
| `device_id` | 目标设备 MAC（与固件上报的 `Device-Id` 一致） |
| `message` | 要播报的文本 |
| `skip_llm` | `true` = 原文直接播报（告警用）；`false` = 经 AI 组织后播报 |
| `auto_listen` | `false` = 播完结束；`true` = 播完自动进入聆听 |

返回结果：

- `{"success":true}` → 设备立即播报该文本；
- `device ... not found or offline` → 设备不在线，检查固件的长连接 / 断线重连。

### 验收标准

设备**待机 5 分钟以上**，调用上面的接口，能**无需任何操作主动开口播报** —— 即为接入成功。
